# 开发日志

## 2026-04-09

### 1. 认证流程优化
- **问题**：注册后自动登录，但未验证邮箱的用户无法再次登录
- **解决方案**：
  - 修改注册流程，注册成功后不再自动登录
  - 添加验证邮箱提示信息
  - 登录时检查邮箱验证状态，显示清晰的错误提示
  - 添加重新发送验证邮件功能
- **修改文件**：`src/App.tsx`

### 2. Supabase 邮箱验证链接修复
- **问题**：Supabase 发送的验证邮件链接指向 localhost:3000
- **解决方案**：
  - 在注册时添加 `redirectTo` 参数，指定验证成功后的重定向 URL
  - 添加 `VITE_SITE_URL` 环境变量
  - 更新 `.env.local` 文件，添加生产环境 URL
- **修改文件**：`src/App.tsx`、`.env.local`

### 3. 登录状态保持问题修复
- **问题**：刷新网页后用户状态丢失，需要重新登录
- **解决方案**：
  - 添加 `supabase.auth.onAuthStateChange` 监听器，实时监听认证状态变化
  - 优化用户数据加载逻辑，确保会话恢复时能正确加载用户数据
  - 添加错误处理，当加载用户数据失败时自动创建新的用户记录
- **修改文件**：`src/App.tsx`

### 4. 移动设备适配优化
- **问题**：手机上今日页面任务的小圆点显示不全，难易度被遮住
- **解决方案**：
  - 修改任务卡片布局，添加 `flex-wrap` 允许元素自动换行
  - 增加容器高度，为内容提供更多显示空间
  - 为所有子元素添加 `flex-shrink-0`，防止被压缩变形
- **修改文件**：`src/App.tsx`

### 5. XP 计算逻辑修复
- **问题**：修改任务难易度和优先级时，计算出来的 XP 值与预期不符
- **解决方案**：
  - 将乘法调整改为加法调整，确保计算结果为整数
  - 困难难度：30 XP
  - 中等难度：20 XP
  - 简单难度：10 XP
  - 高优先级：+10 XP
  - 中优先级：+5 XP
  - 低优先级：+0 XP
- **修改文件**：`src/App.tsx`

### 6. 商店显示优化
- **问题**：商店物品显示消耗 XP，但实际上应该消耗余额
- **解决方案**：
  - 修改商店余额显示，将 "Spendable XP" 改为 "可使用余额"
  - 修改商品价格显示，将 "{item.cost} XP" 改为 "{item.cost} 余额"
  - 修改商品编辑界面，将 "价格 (XP)" 改为 "价格 (余额)"
  - 修改购买记录显示，将 "XP" 改为 "余额"
- **修改文件**：`src/App.tsx`

### 7. 音乐播放问题修复
- **问题**：网页端无法播放音乐
- **解决方案**：
  - 将音频文件从项目根目录移动到 `public` 目录
  - 更新 `playSound` 函数中的音频文件路径，使用绝对路径
  - 确保音频文件可以通过根路径访问
- **修改文件**：`src/App.tsx`

### 8. 任务备注指示功能
